                                                                  四、电梯问题

                                                                  问题描述

                                                                  在日常生活中，当我们等待电梯时，最接近我们所在楼层的电梯会优先来到我们的位置。如果有多部电梯同时处于相同楼层，那么序号最靠前的电梯将优先接我们。请你编写一个程序，找到哪一部电梯将接我们上楼。注意：这栋楼的楼层从第 0 层开始，且一定至少有一部电梯。

                                                                  输入格式

                                                                  第一行包含两个整数 n 和 m，分别表示你当前所在的楼层 n 和电梯的数量 m。
                                                                  第二行包含 m 个整数，表示每部电梯当前所在的楼层。

                                                                  输出格式

                                                                  输出接你上楼的电梯的索引 i（从 0 开始计数）。

                                                                  输入输出样例

                                                                  说明

                                                                  在样例 1 中，你当前在 5 楼，有 4 部电梯，分别位于 1 楼、5 楼、7 楼和 2 楼。最靠近你（且距离为 0 楼层）的电梯是位于 5 楼的第 1 号电梯，因此输出 1。

                                                                  在样例 2 中，你在 1 楼，有 2 部电梯，分别位于 0 楼和 -1 楼。最靠近你的是第 0 号电梯，因此输出 0。

                                                                  提示

                                                                  • 保证输入的 n 和 m 都在 int 类型范围内。

                                                                   

                                                                  五、统计爱学习的人数

                                                                  题目描述

                                                                  小牛学长是校科协培训部的负责人之一，负责给学弟学妹们讲解编程语言。然而，编程语言有很多种，每种语言的学习人数也不相同。小牛学长负责培训第二多人想学的编程语言。但当他看到学弟学妹们想学习编程语言的人数统计表时，感到十分困惑。请你帮助他找出学习人数第二多的编程语言的索引。

                                                                  输入格式

                                                                  • 第一行包含一个正整数 n，表示编程语言的门数。

                                                                  • 第二行包含 n 个整数，第 i 个整数表示第 i 门编程语言的学习人数。

                                                                  输出格式

                                                                  输出一个整数，表示第二多人想学的编程语言的索引（从 0 开始计数）。

                                                                  提示

                                                                  保证至少存在两种学习人数不同的编程语言

                                                                  七、公交车悖论

                                                                  问题背景

                                                                  来源于bilibili up主——毕导

                                                                  毕导常常困惑，为什么公交车明明每10分钟一班，他却总是要等八九分钟，而不是平均的5分钟，难道他是“先天等公交圣体”？这背后其实隐藏了著名的“公交车悖论”：即使公交车平均每10分钟一班，乘客的平均等待时间也会比5分钟更长。

                                                                  举个例子，如果一小时内有六班公交车，理想情况下每10分钟一班，乘客随机到达车站，平均等待时间是5分钟。然而，如果公交车到站间隔不均匀，前三班间隔5分钟，后三班间隔15分钟，六班车的到站时间变成了7:05、7:10、7:15、7:30、7:45和8:00。

                                                                  在这种情况下，乘客在7:00到7:15之间到达车站，平均等待时间是2.5分钟；在7:15到8:00之间到达，平均等待时间是7.5分钟。综合计算，乘客的平均等待时间增加到了6分25秒，比理想情况下的5分钟长了不少。

                                                                  这说明，当公交车到站间隔不均匀时，乘客的平均等待时间会增加，公交车似乎总是比想象中更难等到。

                                                                  问题描述

                                                                  有 n 辆公交车，理想情况下，每辆公交车的发车间隔为 m 分钟，且公交车在理想情况下发车后能瞬移到达乘客所在的公交车站。但由于各种因素影响，每辆公交车会比理想情况推迟 xi 分钟到站。从第一辆公交车的理想出发时间（即 0 分钟）到最后一辆公交车的实际到站时间之间，乘客在时间上随机且均匀地出现在公交站。请计算乘客的期望等待时长（分钟）。

                                                                  输入

                                                                  • 第一行输入两个整数，n 和 m，中间用空格隔开，表示公交车的数量和理想情况下的到站间隔（分钟）

                                                                  • 第二行输入 n 个整数，表示每辆公交车的推迟时间 xi 分钟。

                                                                  输出格式

                                                                  • 输出一个实数，表示乘客的期望等车时长（分钟），结果保留两位小数（四舍五入）。

                                                                  保证条件：

                                                                  • 所有公交车的实际到站时间总体呈增长趋势：Ti+1≥Ti

                                                                  • 实际到站时间不早于0：Ti≥0

                                                                  • 乘客从时间0到最后一辆公交车的实际到站时间随机均匀出现。

                                                                  • 可以推迟负数分钟

                                                                  • 实际到站时间可以早于理想到站时间，但是不会早于上一辆公交车到站。

                                                                  • 在理想情况下,公交车发车后能瞬间到达公交车站。
